Git Pull

ЧТО ЭТО? Команда которая получает изменения из удаленного репозитория с текущей веткой.

ДЛЯ ЧЕГО? Обновляет локальные ветки с последними изменениями.

КАК РАБОТАЕТ? Делает git fetch и git merge

Git Merge

ЧТО ЭТО? Код который обьединяет две ветки.

ДЛЯ ЧЕГО? Поправляет изменения с одной ветки в другую.

КАК РАБОТАЕТ? Объединяет коммиты и создает новый коммит.

WORKFLOW

ЧТО ЭТО? Легкая последовательность дальнейшей разработки.

ДЛЯ ЧЕГО? Чтобы было легче делать разработки более удобным для нас способом.

КАК РАБОТАЕТ? Делаем основные шаги, которые обязательно выполнить, используя разные вспомогательные инструменты

Git Rebase

Git rebase - это команда которую мы используем для того, чтобы перенести коммит из одной ветки в другую. Как работает? Он берет все коммиты из выбранной ветки и добавляет в новую ветку. Для чего нужен? чтобы бюыло легчу переносить коммиты из одной ветки в другую

Git Fetch

Git fetch - эта команда которая позволяет загрузить коммиты, файлы и ссылки из другого репозитория в ваш собственный репозиторий Как работает? Он загружает изменения (коммиты, ветки, ссылки) из другого репозитория, но не объединяет их с вашей текущей веткой. Для чего? чтобы получить информации о другом репозитории
